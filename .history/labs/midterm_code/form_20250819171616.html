ตัวอย่างฟอร์ม: action / method / GET / POST

สรุปเร็ว:
• action = URL ปลายทางที่จะส่งฟอร์มเมื่อกด submit (ถ้าไม่ใส่ → ส่งมาหน้าเดิม)
• method = วิธี HTTP (ค่าเริ่มต้นคือ GET / ระบุ POST สำหรับข้อมูลส่วนตัวหรือข้อมูลจำนวนมาก)
• GET: ข้อมูลจะโชว์บน URL (เหมาะกับการค้นหา/แชร์ลิงก์)
• POST: ข้อมูลอยู่ใน request body (เหมาะกับข้อมูลลับ/ไฟล์/ฟอร์มยาว)

ค้นหา (GET)

ฟอร์มนี้ตั้ง method="get" และ ไม่ใส่ action ⇒ เบราว์เซอร์จะส่งมาที่หน้าเดิมพร้อมพารามิเตอร์บน URL
หลังส่งแล้ว สังเกตดูที่แถบที่อยู่ (จะเห็นเช่น ?q=...&tags[]=frontend)

สมัครสมาชิก (POST)

ฟอร์มนี้ตั้ง method="post" และ enctype="multipart/form-data" (กรณีมีไฟล์)
เดโมนี้ “ดัก submit” ด้วย JS เพื่อแสดง payload ที่จะส่งออกมาให้เห็นในหน้า (โดยไม่ต้องมีเซิร์ฟเวอร์)
งานจริงให้ตั้ง action เป็น URL ของปลายทาง เช่น /users/register

เคล็ดลับเพิ่มเติม:
• ฟอร์มที่อัปโหลดไฟล์ → ต้องใช้ method="post" และ enctype="multipart/form-data"
• ข้อมูลที่เป็นความลับ (เช่น รหัสผ่าน) → ใช้ POST เท่านั้น
• ต้องการให้เลือกได้ทีละตัว (radio) → ตั้ง name เดียวกันในทุกตัวเลือก
• ต้องการส่งหลายค่า (checkbox หลายตัว) → ใช้ชื่อแบบ tags[] เพื่อรวมเป็นชุด
